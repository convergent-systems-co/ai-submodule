Layer 1

Intent

Captures what needs to be done via issues, design intents, and feature requests.

Layer 2

Cognitive

AI personas analyze intent and plan implementation using policy profiles.

Layer 3

Execution

Coder agents implement changes in isolated worktrees.

Layer 4

Runtime

CI/CD pipeline, PR monitoring, governance approval, and merge.

Layer 5

Evolution

Metrics collection, drift detection, and policy tuning.

Intent Layer

  • GitHub Issues with structured acceptance criteria
  • Design Intents (DIs) for architectural changes
  • Feature requests from stakeholders
  • Validated by Tech Lead for completeness

Agent Protocol

Cognitive Layer

  • 7 Personas: Project Manager, Tech Lead, DevOps Engineer, Coder, IaC Engineer, Test Evaluator, Documentation Reviewer
  • 21 Review Panels: Code review, security, documentation, cost, data governance, and more
  • Policy Engine: Deterministic YAML-based policy evaluation
  • 4 Policy Profiles: default, fin_pii_high, infrastructure_critical, fast_track

Persona Definitions | Policy Profiles

Execution Layer

  • Parallel Dispatch: Up to N Coder agents work simultaneously
  • Git Worktrees: Each agent gets an isolated worktree
  • Test Coverage Gate: 80% minimum before push
  • Tester Validation: Independent evaluation of each implementation

Coder Persona | Tester Persona

Runtime Layer

  • GitHub Actions: dark-factory-governance.yml workflow
  • PR Monitoring: CI status, Copilot recommendations, review threads
  • Governance Approval: github-actions[bot] approves when policy passes
  • Merge Decision: Auto-merge, escalation, or block

Workflow Files

Evolution Layer

  • Metrics: Panel effectiveness, review cycle counts, time-to-merge
  • Drift Detection: Schema evolution tracking
  • Policy Tuning: Threshold adjustments based on observed data
  • Retrospectives: Aggregated session analysis
